## Задание
16.	Система оценки безопасности сети
Создание экспертной системы для анализа сетевых угроз.

### Требование к отчету

#### 1.	Описание предметной области
•	Анализ специфики предметной области 
•	Основные понятия и термины
•	Обзор и анализ близких технических решений (систем-аналогов). Не менее 3-х. Результаты анализа должны быть представлены в виде таблицы сравнения.
•	Источники информации (не менее 10 ссылок на литературу)
•	Объем: 3-4 страницы
#### 2.	База знаний
•	Не менее 50 продукционных правил
•	Структурированное представление
•	Примеры фактов и правил
#### 3.	Проверка непротиворечивости
•	Методы проверки
•	Результаты анализа
•	Исправления при необходимости (привести не менее 3-х пар противоречивых правил (у которых условия совпадают или пересекаются, а действия противоречат))
#### 4.	Средства разработки
•	Обоснование выбора языка программирования
•	Описание используемых библиотек
•	Инструменты разработки
#### 5.	Техническое решение
•	Архитектура системы
•	Схема работы
•	Алгоритмы реализации
#### 6.	Тестирование прототипа
•	Результаты прямого вывода (не менее 2-ух примеров, содержащие исходные факты и детальное описание цепочек вывода (не менее 51 шага и 2-ух проходов))
•	Результаты обратного вывода (не менее 2-ух примеров, содержащие исходные факты и детальное описание цепочек вывода (не менее 51 шага и 2-ух проходов))
•	Анализ производительности (времени прямого и обратного вывода)
#### 7.	Выводы
•	Достигнутые результаты
•	Ограничения системы
•	Перспективы развития
#### 8.	Список литературы
•	Использованные источники
•	Нормативная документация
#### 9.	Программный код
•	Структурированные исходники
•	Комментарии к ключевым участкам
•	Документация по использованию
•	Ссылка на проект или Архив проекта
