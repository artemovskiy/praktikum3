# Инструкция: как повторить анализ работы ЭС

## Цель

Повторно выполнить запуск экспертной системы в CLIPS, сохранить артефакты запуска и оформить разбор результата в формате, принятом в проекте.

## Шаг 0. Обязательный preflight (по AGENTS.md)

Перед началом анализа обязательно:
- прочитать `README.md`;
- изучить материалы по CLIPS (`notes/sreda_clips_extracted.txt` или PDF из `docs/источники/`);
- использовать относительные пути по умолчанию.

Контрольный чек:
- [ ] `README.md` прочитан.
- [ ] Материал по CLIPS изучен.
- [ ] Используются относительные пути.
- [ ] Реализация начинается только после выполнения пунктов выше.

## Шаг 1. Перейти в каталог проекта

```bash
cd path/to/praktikum3
```

Примечание: после перехода в корень проекта в дальнейших командах использовать относительные пути.

## Шаг 2. Запустить МЛВ и сохранить сырой вывод

```bash
printf "(clear)\n(load \"./network_security_assessment.clp\")\n(reset)\n(watch rules)\n(run)\n(facts)\n(exit)\n" | clips > ./notes/mlv_run_example_output.txt
```

Что делает команда:
- очищает среду CLIPS;
- загружает базу знаний;
- выполняет вывод с трассировкой правил;
- печатает итоговые факты;
- сохраняет весь вывод в файл.

## Шаг 3. Проверить базовые критерии корректности запуска

В сохраненном выводе должны быть:
- `TRUE` после загрузки;
- строки `FIRE ...` (трассировка срабатываний правил);
- строка `For a total of ... facts.`;
- финальный факт `(context (key system-state) (value stable))`.

Быстрая проверка:

```bash
rg -n "TRUE|^FIRE|For a total of|system-state" ./notes/mlv_run_example_output.txt
```

## Шаг 4. Проверить соответствие требованиям README

1. Проверить количество правил (должно быть не менее 50):

```bash
rg -n "^\(defrule\s" ./network_security_assessment.clp | wc -l
```

2. Проверить длину цепочки прямого вывода в данном прогоне (ориентир: не менее 51 шага):

```bash
rg -n "^FIRE" ./notes/mlv_run_example_output.txt | wc -l
```

3. Зафиксировать статус:
- что выполнено;
- что еще нужно для полного отчета (например, дополнительные прогоны прямого/обратного вывода).

## Шаг 5. Обновить файл с примером разбора

Целевой файл:
- `./notes/mlv_run_example.md`

Что обновлять:
- блок "Вывод МЛВ (как есть)" заменить актуальным выводом;
- раздел с комментариями оставить с явной пометкой:
  "это мои комментарии, не часть результата работы МЛВ";
- разбор фактов по группам:
  - указать категорию группы (`входные данные` / `промежуточные факты` / `итоговый результат анализа сети`);
  - приложить дословный `code block` с фактами группы;
  - добавить общий комментарий;
  - при необходимости добавить таблицу с пояснениями ключевых фактов.

## Шаг 6. Обновить текущий статус

Целевой файл:
- `./notes/current_es_status.md`

Что обновлять:
- результаты последней проверки (число срабатываний, число фактов, финальный статус);
- пункт о соответствии заданию;
- ссылка на актуальный файл с примером запуска.

## Шаг 7. Обновить индекс заметок

Файл:
- `./notes/README.md`

Что сделать:
- добавить/обновить пункт о выполненном прогоне и обновленных файлах, чтобы коллеги видели актуальный статус.
